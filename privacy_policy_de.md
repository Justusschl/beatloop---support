Diese Datenschutzerklärung erläutert, wie wir Ihre personenbezogenen Daten erheben, verwenden, verarbeiten und schützen, wenn Sie unsere mobile Anwendung BeatLoop nutzen. ## 5. Integrationen von Musikdiensten Dritter

BeatLoop integriert die folgenden Musikdienste von Drittanbietern, um Nutzern den Zugriff auf und das Üben mit Musik zu ermöglichen. Jede Integration umfasst eine bestimmte Datenerhebung und -verarbeitung:

### 5.1 YouTube (YouTube API Services)

BeatLoop verwendet YouTube API Services, um Nutzern die Suche nach und Wiedergabe von Musik von YouTube zu ermöglichen.

- **Erhobene Daten:** Video-Metadaten (Titel, Künstler, Dauer), Wiedergabe-Interaktionsdaten
- **NICHT erhobene Daten:** BeatLoop lädt KEINE YouTube-Audio- oder Videoinhalte herunter, speichert oder zwischenspeichert sie. Die gesamte Wiedergabe erfolgt über die YouTube-API.
- **YouTube-Nutzungsbedingungen:** [https://www.youtube.com/t/terms](https://www.youtube.com/t/terms)
- **Google-Datenschutzerklärung:** [https://policies.google.com/privacy](https://policies.google.com/privacy)

**Einhaltung der Google API Services User Data Policy:** Die Nutzung der von YouTube API Services erhaltenen Informationen durch BeatLoop entspricht der [Google API Services User Data Policy](https://developers.google.com/terms/api-services-user-data-policy), einschließlich der Limited-Use-Anforderungen.

Nutzer können den Zugriff von BeatLoop auf ihre YouTube-Daten über die Google-Sicherheitseinstellungsseite unter [https://security.google.com/settings/security/permissions](https://security.google.com/settings/security/permissions) widerrufen.

### 5.2 Apple Music (MusicKit)

BeatLoop verwendet die Apple MusicKit API, um Nutzern die Suche nach und das Streamen von Musik von Apple Music zu ermöglichen.

- **Erhobene Daten:** Titel-Metadaten (Titel, Künstler, Album, Dauer), Bibliothekszugriff (mit Erlaubnis des Nutzers)
- **NICHT erhobene Daten:** BeatLoop lädt KEINE Apple Music-Audioinhalte herunter, speichert oder zwischenspeichert sie. Audio wird über Apple Music gestreamt.
- **Voraussetzung:** Nutzer müssen über ein aktives Apple Music-Abonnement verfügen, um diese Funktion nutzen zu können.
- **Apple Media Services Terms and Conditions:** [https://www.apple.com/legal/internet-services/itunes/](https://www.apple.com/legal/internet-services/itunes/)

### 5.3 Spotify (Spotify Web API)

BeatLoop verwendet die Spotify Web API, um Nutzern die Suche nach und Wiedergabe von Musik von Spotify zu ermöglichen.

- **Erhobene Daten:** Titel-Metadaten (Titel, Künstler, Album, Dauer), Wiedergabestatus
- **NICHT erhobene Daten:** BeatLoop lädt KEINE Spotify-Audioinhalte herunter, speichert oder zwischenspeichert sie. Audio wird über Spotify gestreamt.
- **Voraussetzung:** Nutzer müssen über ein aktives Spotify-Konto verfügen, um diese Funktion nutzen zu können.
- **Spotify-Nutzungsbedingungen:** [https://www.spotify.com/legal/end-user-agreement/](https://www.spotify.com/legal/end-user-agreement/)
- **Spotify-Datenschutzerklärung:** [https://www.spotify.com/legal/privacy-policy/](https://www.spotify.com/legal/privacy-policy/)

### 5.4 Lokale Dateien

Nutzer können Audiodateien aus dem lokalen Speicher ihres Geräts importieren.

- Lokale Dateien werden vollständig auf dem Gerät verarbeitet und niemals auf unsere Server hochgeladen.
- BeatLoop überwacht, scannt oder validiert den Inhalt oder Urheberrechtsstatus lokaler Dateien nicht.

### 5.5 Google Drive

BeatLoop ermöglicht es Nutzern, Audiodateien aus ihrem Google Drive zu importieren.

- **Zugriffsart:** Schreibgeschützter Zugriff auf Dateien, die der Nutzer ausdrücklich über den Google Picker auswählt
- **Erhobene Daten:** Dateimetadaten (Name, Typ, Größe) und Audiodateiinhalt für die Wiedergabe
- **Datenspeicherung:** Importiertes Audio wird ausschließlich lokal auf dem Gerät des Nutzers verarbeitet und gespeichert. BeatLoop lädt KEINE Google Drive-Dateien auf seine Server hoch, zwischenspeichert oder speichert sie nicht
- **Datenweitergabe:** Google Drive-Dateidaten werden nicht an Dritte weitergegeben
- **Widerruf:** Nutzer können den Zugriff von BeatLoop jederzeit über [https://myaccount.google.com/permissions](https://myaccount.google.com/permissions) widerrufen
- **Einhaltung der Google API Services User Data Policy:** Die Nutzung der von Google APIs erhaltenen Informationen durch BeatLoop entspricht der [Google API Services User Data Policy](https://developers.google.com/terms/api-services-user-data-policy), einschließlich der Limited-Use-Anforderungen.
- **Google-Nutzungsbedingungen:** [https://developers.google.com/terms](https://developers.google.com/terms)

### 5.6 Speicherung von Metadaten

Metadaten von Streaming-Musik (Songtitel, Künstler, Tempo-Markierungen, Loop-Punkte) können zu Zwecken der Sitzungsverwaltung lokal auf Ihrem Gerät gespeichert werden. Die tatsächlichen Audioinhalte von Streaming-Diensten werden von BeatLoop NICHT gespeichert.
